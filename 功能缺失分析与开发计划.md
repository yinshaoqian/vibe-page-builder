## 一、搭建器（Builder.vue）— 缺失清单（417 行，45%）

### 1.1 布局容器的嵌套功能（P0）
**现状**：布局容器（栅格/弹性/卡片）拖到画布后只显示图标占位，没有任何子元素承载能力。

**需要实现**：
- [ ] 栅格容器：拖入后可设置 `cols`（2/3/4 列），子模块自动分配到各列
- [ ] 弹性容器：支持 `direction`（row/column）、`wrap`、`justify`、`align`
- [ ] 卡片容器：带 header/body 区域，可拖入子模块
- [ ] 拖拽一个模块到布局容器上时 → 自动成为其子元素
- [ ] 嵌套层级指示（缩进线 / 面包屑导航）

**数据结构**：
```typescript
interface CanvasItem {
  instanceId: string
  componentId: string
  name: string
  type: 'layout' | 'module'
  layoutType?: 'grid' | 'flex' | 'card'
  children?: CanvasItem[]
  props?: Record<string, any>
}
```

### 1.2 画布配置持久化（P0）
**现状**：`canvasItems` 仅在内存中，刷新丢失。

**需要实现**：
- [ ] 页面打开时通过 postMessage 向主平台请求保存的配置
- [ ] 保存时序列化整个 `canvasItems` 树（含嵌套结构）
- [ ] 添加「另存为新页面」「加载页面」功能
- [ ] 页面配置数据结构

### 1.3 拖拽体验增强（P1）
- [ ] 拖拽过程中显示插入位置指示器（水平线 + 箭头）
- [ ] 拖到布局容器上时高亮容器边缘
- [ ] 按住 `Ctrl` 拖拽 = 复制
- [ ] 右键菜单（编辑属性 / 复制 / 删除 / 移到容器）

### 1.4 撤销 / 重做（P1）
- [ ] 维护操作历史栈
- [ ] `Ctrl+Z` / `Ctrl+Shift+Z` 快捷键
- [ ] 工具栏撤销/重做按钮

### 1.5 导出页面为 JSON / HTML（P1）
- [ ] 「导出配置」→ 页面描述 JSON
- [ ] 「导出页面」→ 独立 HTML 文件
- [ ] 可用于预览或部署

### 1.6 响应式配置（P2）
- [ ] 每个设备视图可独立调整组件顺序
- [ ] 或指定组件在某个设备下隐藏

## 二、组件编辑器（ComponentEditor.vue）— 缺失清单（829 行，75%）

### 本轮已完成（v1 → v2 新增功能）
- ✅ **沙箱预览作为主视图** — 默认展示沙箱，不再需要手动「粘贴 → 预览」
- ✅ **代码编辑功能** — 新增 SFC 代码编辑区，右上角「预览」按钮
- ✅ **自动提取 $t('XXX')** — 粘贴代码时自动解析所有 `$t('key')`，填充到多语言面板
- ✅ **自动提取颜色** — 解析 hex / Tailwind color classes / rgba，分类展示到颜色调整面板
- ✅ **颜色分组展示** — Hex 颜色 / 文字色 / 背景色 / 边框色 / 其他色，每组带色块预览 + 颜色选择器
- ✅ **事件绑定选取器** — 沙箱中 hover 高亮、click 选定元素 → 绑定 @click/@dblclick
- ✅ **导出弹窗** — 完整的导出代码预览 + 复制 + 下载 .vue 文件
- ✅ **组件文件管理** — 新建/加载/保存/删除，localStorage 持久化，postMessage 同步
- ✅ 内置 `extractScript` 函数（new RegExp 方式避免 SFC 解析器误判）

### 2.1 与全局配置的颜色联动（P0）
**现状**：颜色面板独立显示预设色，导出的 .vue 代码写死 `#2F6BFF`。

**需要实现**：
- [ ] 颜色面板读取 `GlobalConfig.themeColors` 的 7 个预设色
- [ ] 用户调整颜色后 → 更新到 CSS 变量引用
- [ ] 导出的 .vue 改为 `var(--color-primary)` 等 CSS 变量
- [ ] 导出时附带 GlobalConfig 的 CSS 变量定义

### 2.2 沙箱预览响应语言切换（P0）
**现状**：沙箱 `$t()` 只返回 key 本身，切换语言 Tab 不会更新预览。

**需要实现**：
- [ ] 沙箱 `$t()` 根据当前活跃语言返回对应翻译值
- [ ] 切换语言 Tab → 触发重渲染

### 2.3 AI 翻译对接（P1）
- [ ] **替换占位按钮**（当前 `alert('AI 一键翻译功能对接中')`）
- [ ] 支持「翻译当前条目」「翻译全部」两种模式
- [ ] 翻译后自动填入对应语言输入框

### 2.4 版本历史 / 草稿管理（P1）
- [ ] 每次「保存」生成新版本
- [ ] 文件列表中显示版本数，可展开查看历史
- [ ] 回滚到任意历史版本

### 2.5 导入 .vue 文件（P1）
- [ ] 「从文件导入」按钮 → 文件选择器
- [ ] 解析上传的 .vue → 自动填充所有字段

### 2.6 模板语法高亮（P2）
- [ ] 代码编辑区增加语法高亮

## 四、跨模块与工程 — 缺失清单

### 4.1 postMessage 完整生命周期（P0）
**现状**：`usePostMessage.ts` 只支持基础 send/receive，无协议落地。

**需要实现**（已在对接说明书中定义）：
- [ ] App.vue `onMounted` 时发 `ready` 消息
- [ ] `save-page-config` / `save-component` / `list-components` / `delete-component` / `load-global-config` / `save-global-config` 6 条消息
- [ ] 统一错误响应格式

### 4.2 统一加载状态与错误处理（P1）
- [ ] 所有异步操作增加 loading spinner
- [ ] 错误消息统一展示（toast / banner）
- [ ] 网络超时重试机制

### 4.3 弹窗/通知系统（P1）
**现状**：多处仍使用 `alert()` / `confirm()`（ComponentEditor 的 `deleteFile`、`fillAllLanguages` 等）

- [ ] 替换为自定义弹窗组件
- [ ] 成功/失败/信息三种通知类型

### 4.4 沙箱性能优化（P2）
- [ ] N 个模块同时渲染时，每个创建独立 `createApp`，内存开销大
- [ ] 不可见组件不做实时沙箱渲染，只显示静态截图
- [ ] 画布模块只渲染轻量预览，全功能预览统一在右侧面板

### 4.5 TypeScript 完善（P2）
- [ ] 当前仍有少量 `as any`（沙箱 componentDef、postMessage 事件 data）
- [ ] 沙箱 Mock 函数签名未完全对齐

### 4.6 单元测试（P3）
- [ ] 核心 composable 测试（usePostMessage, useComponentStore）
- [ ] parseSFC / extractI18nKeys / extractColors 工具测试
