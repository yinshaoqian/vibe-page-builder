# 可视化页面搭建器 — 平台对接说明书

> **架构升级 v2**：搭建器子应用通过 `postMessage` 接收主平台的鉴权与上下文信息后，**直接与后端 API 通信**，不再通过主平台代理存储请求。  
> 主平台只负责：① 初始化时传入身份与上下文 ② 接收搭建器抛出的事件/导航请求。

---

## 一、架构总览

### 1.1 通讯方式

| 通讯方向 | 方式 | 用途 |
|----------|------|------|
| **主平台 → 子应用** | `postMessage` | 初始化时下发 `token`/`refreshToken`/`currentOrgId`/`appName` |
| **子应用 → 后端 API** | HTTP (fetch/axios) | 所有数据 CRUD（页面配置、组件文件、全局配置） |
| **子应用 → 主平台** | `postMessage` | 事件通知（导航、错误、权限不足等） |

### 1.2 数据流示意

```
┌───────────────────┐         postMessage (初始化)        ┌───────────────────┐
│     主平台         │ ─────── { token, refreshToken,     │     子应用         │
│   (iframe 嵌入)    │          currentOrgId, appName }    │   (搭建器/编辑器)   │
│                    │ ◀─────── { type: 'navigate', ... } │                    │
└───────────────────┘         { type: 'error', ... }      └───────┬───────────┘
                                                                   │
                                             ┌─────────────────────┼──────────┐
                                             │  HTTP (Bearer token) │          │
                                             ▼                     ▼          ▼
                                      ┌──────────────┐   ┌──────────┐   ┌──────────┐
                                      │ 页面配置 API   │   │ 组件文件  │   │ 全局配置  │
                                      │ /api/page/*   │   │ /api/comp*│   │ /api/conf*│
                                      └──────────────┘   └──────────┘   └──────────┘
```

## 二、初始化协议（主平台 → 子应用）

### 2.1 子应用挂载时的初始化消息

主平台在子应用加载完成后，**一次性**下发鉴权与上下文信息：

```typescript
// 主平台 → 子应用
interface PlatformInitMessage {
  type: 'init'
  payload: {
    token: string              // 鉴权令牌 (JWT / Bearer Token)
    refreshToken: string       // 刷新令牌
    currentOrgId: string       // 当前组织/租户 ID
    appName: string            // 当前应用名称（如 "Loan Calculator"）
  }
}
```

**时序**：
```
子应用 iframe 加载完成
      │
      ▼
子应用发送 { type: 'ready' }   ← 可选，主平台可监听此事件触发 init
      │
      ▼
主平台发送 { type: 'init', payload: { token, refreshToken, currentOrgId, appName } }
      │
      ▼
子应用存储到内存（非 localStorage）
      │
      ▼
子应用开始调用后端 API
```

### 2.2 token 刷新机制

子应用内置 TokenManager，自动处理 token 过期：

```typescript
interface TokenRefreshMessage {
  type: 'token-refresh'
  payload: {
    token: string              // 新 token
    refreshToken: string       // 新 refreshToken（可能不变）
  }
}
```

- 主平台**不限制推送时机**，可在任意时刻下发新 token
- 子应用收到后自动更新内存中的 token
- token 过期时，子应用用 `refreshToken` 请求新 token（直调后端），若失败则向主平台发送 `{ type: 'auth-error', payload: { reason: 'refresh_failed' } }`

## 三、postMessage 协议表

### 3.1 主平台 → 子应用

| type | payload | 触发时机 | 说明 |
|------|---------|----------|------|
| `init` | `{ token, refreshToken, currentOrgId, appName }` | 子应用加载完成 | **必须下发**，未收到则子应用无法调 API |
| `token-refresh` | `{ token, refreshToken }` | token 即将过期 / 主平台主动刷新 | 可选下发 |
| `navigate` | `{ path: '/builder' }` | 主平台控制子应用内部跳转 | 可选 |

### 3.2 子应用 → 主平台

| type | payload | 说明 |
|------|---------|------|
| `ready` | `void` | 子应用已挂载就绪，主平台可下发 init |
| `navigate` | `{ path: string }` | 子应用请求主平台跳转（如需跳出 iframe） |
| `auth-error` | `{ reason: 'token_expired' \| 'refresh_failed' \| 'unauthorized' }` | 鉴权异常，主平台可触发重新登录 |
| `error` | `{ message: string, code?: string, detail?: any }` | 子应用运行时异常通知 |

> **注意**：所有数据 CRUD（保存/加载页面配置、组件文件、全局配置等）**不再走 postMessage**，改为子应用直调后端 API（见第四章）。

## 四、后端 API 接口规范

### 4.1 通用约定

| 项目 | 规范 |
|------|------|
| Base URL | 由主平台在 `init` 中通过 `payload.baseUrl` 指定，或子应用使用默认值 |
| 鉴权方式 | `Authorization: Bearer {token}` |
| 组织上下文 | Header `X-Org-Id: {currentOrgId}` |
| 请求体 | `Content-Type: application/json` |
| 响应格式 | 统一 `{ success: boolean, data?: any, error?: { code: string, message: string } }` |

### 4.2 页面配置 API

#### `GET /api/page/config`
获取当前页面的画布配置。

**Headers**:
```
Authorization: Bearer {token}
X-Org-Id: {currentOrgId}
```

**Response**:
```json
{
  "success": true,
  "data": {
    "id": "page_xxx",
    "name": "首页",
    "canvasItems": [
      { "instanceId": "item_1", "componentId": "comp_header", "props": {}, "order": 0 },
      { "instanceId": "item_2", "componentId": "comp_card", "props": {}, "order": 1 }
    ],
    "device": "mobile",
    "updatedAt": "2026-01-01T00:00:00Z"
  }
}
```

#### `POST /api/page/config`
保存页面配置。

**Headers**: 同上

**Request Body**:
```json
{
  "id": "page_xxx",
  "canvasItems": [...],
  "device": "mobile"
}
```

### 4.3 组件文件 API

#### `GET /api/components` — 获取组件列表

#### `GET /api/components/:id` — 获取单个组件详情

#### `POST /api/components` — 创建/更新组件

#### `DELETE /api/components/:id` — 删除组件

### 4.4 全局配置 API

#### `GET /api/global-config` — 获取全局配置

#### `POST /api/global-config` — 保存全局配置

> 各 API 的详细 `Request/Response` 结构同原说明书§2 中定义的数据类型（`StoredComponent`、`GlobalConfig`），仅通讯方式从 postMessage 改为 HTTP。

## 六、初始化流程（更新后）

```
子应用 iframe 加载完成
      │
      ▼
Vue 应用挂载
      │
      ▼
发送 { type: 'ready' }  ← 通知主平台子应用就绪
      │
      ▼
等待 { type: 'init', payload: { token, refreshToken, currentOrgId, appName } }
      │   └─ 主平台必须在收到 ready 后尽快下发
      │      超时 10 秒未收到 → 降级 localStorage 模式（调试用）
      ▼
TokenManager.initFromPlatform(msg)
      │
      ├──► 存储 token / refreshToken / currentOrgId / appName
      │
      ▼
ApiClient 初始化（baseURL 从 init.payload 获取或默认值）
      │
      ▼
各模块通过 ApiClient 向后端请求数据：
  ├── Builder.vue       → GET /api/page/config
  ├── ComponentEditor   → GET /api/components
  └── GlobalConfig.vue  → GET /api/global-config
```

## 七、关键类型定义汇总

```typescript
// ================================================================
// 一、postMessage 协议类型（主平台 ↔ 子应用）
// ================================================================

// 主平台 → 子应用
interface PlatformToChildMessage {
  type: 'init' | 'token-refresh' | 'navigate'
  payload: InitPayload | TokenRefreshPayload | NavigatePayload
}

interface InitPayload {
  token: string                    // 鉴权令牌
  refreshToken: string             // 刷新令牌
  currentOrgId: string             // 当前组织/租户 ID
  appName: string                  // 应用名称
  baseUrl?: string                 // 后端 API 基地址（可选，默认由子应用配置）
}

interface TokenRefreshPayload {
  token: string
  refreshToken: string
}

interface NavigatePayload {
  path: string                     // 子应用内部路由路径
}

// 子应用 → 主平台
interface ChildToPlatformMessage {
  type: 'ready' | 'navigate' | 'auth-error' | 'error'
  payload?: ReadyPayload | NavigatePayload | AuthErrorPayload | ErrorPayload
}

interface ReadyPayload {}           // 无数据

interface AuthErrorPayload {
  reason: 'token_expired' | 'refresh_failed' | 'unauthorized'
}

interface ErrorPayload {
  message: string
  code?: string
  detail?: any
}

// ================================================================
// 二、后端 API 通用响应格式
// ================================================================

interface ApiResponse<T = any> {
  success: boolean
  data?: T
  error?: {
    code: string                   // 错误码，如 'UNAUTHORIZED', 'NOT_FOUND'
    message: string                // 人类可读描述
  }
}

// ================================================================
// 三、业务数据模型（与 v1 一致，仅通信方式改为 HTTP）
// ================================================================

// === 页面配置 ===
interface PageConfig {
  id: string
  name: string
  canvasItems: CanvasItem[]
  device: 'desktop' | 'tablet' | 'mobile'
  updatedAt: string
}

interface CanvasItem {
  instanceId: string
  type: 'module' | 'container'
  componentId?: string
  containerType?: 'grid' | 'flex' | 'card'
  props?: Record<string, any>
  order: number
}

// === 组件文件 ===
interface StoredComponent {
  id: string
  name: string
  description: string
  version: string
  updatedAt: string
  template: string
  script: string
  style: string
  i18nEntries: I18nEntry[]
  colorVars: ColorVar[]
  bindings: EventBinding[]
  eventBusName: string
}

interface I18nEntry {
  key: string
  translations: Record<string, string>
}

interface ColorVar {
  name: string
  label: string
  value: string
}

interface EventBinding {
  id: string
  elementId: string
  selector: string
  event: 'click' | 'dblclick'
  action: string
  payload?: string
}

// === 全局配置 ===
interface GlobalConfig {
  id: string
  name: string
  updatedAt: string
  colors: {
    primary: string
    secondary: string
    textPrimary: string
    textSecondary: string
    accent1: string
    accent2: string
    accent3: string
    accent4: string
  }
  languages: Language[]
  i18nEntries: GlobalI18nEntry[]
}

interface Language {
  code: string
  name: string
  isEnabled: boolean
  isDefault?: boolean
}

interface GlobalI18nEntry {
  key: string
  translations: Record<string, string>
}

// ================================================================
// 四、子应用内部模块类型
// ================================================================

export class TokenManager {
  private token: string
  private refreshToken: string
  private currentOrgId: string
  private appName: string

  /** 收到主平台 init 消息时调用 */
  initFromPlatform(payload: InitPayload): void

  /** 收到 token-refresh 消息时调用 */
  refresh(newToken: string, newRefreshToken: string): void

  /** 构建 API 请求头 */
  getAuthHeaders(): { Authorization: string; 'X-Org-Id': string }

  /** 获取当前 token */
  getToken(): string

  /** 获取当前 orgId */
  getOrgId(): string

  /** 获取应用名 */
  getAppName(): string
}
```

## 八、开发调试说明

### 8.1 无主平台时调试

子应用支持降级模式。当 `TokenManager` 在 10 秒内未收到 `init` 消息时：

1. 使用 `localStorage` 中的模拟 token/orgId（开发环境预置）
2. `ApiClient` 请求本地 Mock 地址或直接读 `localStorage`
3. `useComponentStore` 自动降级到 localStorage 存储（同旧版行为）

### 8.2 模拟主平台初始化

```javascript
// 在子应用 iframe 的控制台中模拟主平台下发 init
window.postMessage({
  type: 'init',
  payload: {
    token: 'dev-token-xxx',
    refreshToken: 'dev-refresh-xxx',
    currentOrgId: 'org_dev_001',
    appName: 'Loan Calculator',
    baseUrl: 'http://localhost:5000/api'
  }
}, '*')

// 模拟 token 刷新
window.postMessage({
  type: 'token-refresh',
  payload: {
    token: 'new-token-yyy',
    refreshToken: 'new-refresh-yyy'
  }
}, '*')
```

### 8.3 API 开发调试

后端 API 开发期间，建议实现一个简单的 mock server：

```bash
# 示例：使用 json-server 快速搭建
npx json-server --watch db.json --port 4000
```

在 `init` 的 `baseUrl` 中指向 mock 地址即可。
