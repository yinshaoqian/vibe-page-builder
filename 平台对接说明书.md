# 可视化页面搭建器 — 平台对接说明书

> 本搭建器（以下简称「子应用」）以 `<iframe>` 或 Web Component 方式嵌入主平台，双方通过 **postMessage** 进行双向通讯。  
> 主平台负责所有持久化存储与后端逻辑，子应用只做 UI 渲染与事件收集。

---

## 一、通讯协议总览

### 1.1 基础约定

| 项目 | 规范 |
|------|------|
| 通讯方式 | `window.parent.postMessage()` / `window.addEventListener('message')` |
| 数据格式 | JSON |
| 消息类型字段 | `type`（字符串） |
| 安全校验 | 子应用校验 `event.origin`；主平台校验 `x-session` Header |
| 消息方向 | 子应用 → 主平台（请求动作）；主平台 → 子应用（下发数据/响应） |

### 1.2 消息结构

```typescript
// 子应用 → 主平台（请求）
interface ChildToParentMessage {
  type: string        // 消息类型，见下方协议表
  payload?: any       // 请求数据
  requestId?: string  // 请求 ID，用于异步回包关联（由子应用生成）
}

// 主平台 → 子应用（响应/推送）
interface ParentToChildMessage {
  type: 'platform-data' | 'platform-response'
  payload: {
    action: string                // 对应的请求类型
    data?: any                    // 返回数据
    error?: string                // 错误消息
    requestId?: string            // 关联的请求 ID
  }
}
```

## 二、通讯协议表（postMessage 事件列表）

### 2.1 页面配置（整体布局）

> **说明**：子应用搭建的画布布局，包含放置的组件、位置、顺序等信息。

| 方向 | type | payload | 说明 |
|------|------|---------|------|
| 子→主 | `save-page-config` | `PageConfig` | 保存当前画布布局 |
| 主→子 | `platform-data` | `{ action: 'load-page-config', data: PageConfig }` | 下发已保存的布局 |
| 子→主 | `request-data` | `{ key: 'page-config' }` | 请求加载布局 |

**`PageConfig` 数据结构**：

```typescript
interface PageConfig {
  id: string                    // 页面唯一 ID
  name: string                  // 页面名称
  canvasItems: CanvasItem[]     // 画布上的组件列表（有序）
  device: 'desktop' | 'tablet' | 'mobile'  // 当前编辑设备视图
  updatedAt: string             // 更新时间 ISO 字符串
}

interface CanvasItem {
  instanceId: string            // 实例 ID（唯一标识画布上该位置）
  type: 'module' | 'container'  // 模块组件 | 布局容器
  componentId?: string          // 模块组件的 componentId（type=module 时必填）
  containerType?: 'grid' | 'flex' | 'card'  // 容器类型（type=container 时必填）
  props?: Record<string, any>   // 组件实例化参数（如 { payload: {} }）
  order: number                 // 在画布中的顺序（从 0 递增）
}
```

### 2.2 组件文件管理（组件模块）

> **说明**：ComponentEditor 中创建/编辑的 .vue 组件文件，存储到主平台。

| 方向 | type | payload | 说明 |
|------|------|---------|------|
| 子→主 | `save-component` | `StoredComponent` | 保存/更新一个组件文件 |
| 子→主 | `delete-component` | `{ id: string }` | 删除组件文件 |
| 子→主 | `list-components` | `void` | 请求组件文件列表 |
| 主→子 | `platform-data` | `{ action: 'list-components', data: StoredComponent[] }` | 下发组件列表 |
| 子→主 | `request-data` | `{ key: 'components' }` | 请求加载所有组件 |

**`StoredComponent` 数据结构**：

```typescript
interface StoredComponent {
  id: string                    // 组件唯一 ID
  name: string                  // 组件名（如 "Theme3LoanCalculator"）
  description: string           // 描述文字
  version: string               // 版本号（如 "1.0.0"）
  updatedAt: string             // ISO 时间戳

  // === SFC 三要素（完整组件代码） ===
  template: string              // <template> 内容
  script: string                // <script> 内容（不含 <script> 标签）
  style: string                 // <style> 内容（不含 <style> 标签）

  // === 编辑器数据 ===
  i18nEntries: I18nEntry[]     // 多语言条目
  colorVars: ColorVar[]        // 组件内颜色引用
  bindings: EventBinding[]     // 事件绑定列表
  eventBusName: string          // EventBus 实例名（如 "$EventBus"）
}

interface I18nEntry {
  key: string                   // 翻译 key（如 "选择额度"）
  translations: Record<string, string>  // { zh: "选择额度", en: "Loan Amount", th: "วงเงิน", vn: "Số tiền", ph: "Halaga" }
}

interface ColorVar {
  name: string                  // 变量名（如 "--color-primary"）
  label: string                 // 显示名（如 "主色"）
  value: string                 // 当前值（如 "#2F6BFF"）
}

interface EventBinding {
  id: string                    // 绑定唯一 ID
  elementId: string             // 目标元素注入的 data-warden-id
  selector: string              // CSS selector 或层级路径描述
  event: 'click' | 'dblclick'  // 事件类型
  action: string                // EventBus 动作名（如 "warden-action"）
  payload?: string              // 固定参数 JSON
}
```

### 2.3 全局配置（主题色 + 多语言管理）

> **说明**：GlobalConfig 页面管理的全局主题色和语言配置。

| 方向 | type | payload | 说明 |
|------|------|---------|------|
| 子→主 | `save-global-config` | `GlobalConfig` | 保存全局配置 |
| 子→主 | `request-data` | `{ key: 'global-config' }` | 请求加载配置 |
| 主→子 | `platform-data` | `{ action: 'load-global-config', data: GlobalConfig }` | 下发配置 |

**`GlobalConfig` 数据结构**：

```typescript
interface GlobalConfig {
  id: string
  name: string                    // 配置名称
  updatedAt: string

  // === 主题色：7 个色值 ===
  colors: {
    primary: string               // 品牌主色（默认 #2F6BFF）
    secondary: string             // 次要色（默认 #5B8DEF）
    textPrimary: string           // 主要文字（默认 #152033）
    textSecondary: string         // 次要文字（默认 #637089）
    accent1: string               // 附属色 1
    accent2: string               // 附属色 2
    accent3: string               // 附属色 3
    accent4: string               // 附属色 4
  }

  // === 多语言 ===
  languages: Language[]

  // === i18n 翻译条目 ===
  i18nEntries: GlobalI18nEntry[]
}

interface Language {
  code: string                    // 语言代码，如 "zh" "en" "vn" "th" "ph"
  name: string                    // 语言名，如 "中文" "English"
  isEnabled: boolean              // 是否启用
  isDefault?: boolean             // 是否是默认语言
}

interface GlobalI18nEntry {
  key: string
  translations: Record<string, string>  // 各语言翻译值
}
```

### 2.4 请求初始化数据

> 子应用加载时，一次性请求所有必要数据。

**请求**：
```typescript
{
  type: 'request-data',
  payload: { keys: ['page-config', 'components', 'global-config'] }
}
```

**响应**：
```typescript
{
  type: 'platform-data',
  payload: {
    action: 'init',
    data: {
      'page-config': PageConfig | null,
      'components': StoredComponent[],
      'global-config': GlobalConfig | null
    }
  }
}
```

## 四、初始化流程

```
子应用加载
    │
    ▼
创建 Vue 应用 + 挂载
    │
    ▼
mounted() → 调用 init()
    │
    ├──► 发送 { type: 'request-data', keys: [...] }
    │         │
    │         ▼
    │    主平台返回 { action: 'init', data }

    ├──► 无数据 → 使用 localStorage 默认值（开发调试用）
    │
    ▼
渲染各页面视图
```

## 五、导出组件格式

> ComponentEditor 的「导出」功能生成的 .vue 文件格式。

```vue
<template>
  <!-- 组件模板内容（含事件绑定） -->
</template>

<script>
// 多语言：通过 $i18n.local 切换
// 事件：通过 $EventBus.$emit('action-name', payload)
// 主题色：通过 CSS 变量 --color-primary 等

export default {
  name: 'ExportedComponent',
  mounted() {
    this.$root.$on('language-changed', (lang) => {
      this.$i18n.locale = lang
    })
  },
  methods: {
    // 事件绑定的方法
    handleWardenAction() {
      this.$EventBus.$emit('warden-action', { ... })
    }
  }
}
</script>

<style scoped>
/* 使用 var(--color-primary) 引用主题色 */
</style>
```

## 七、开发调试说明

### 7.1 无主平台时调试

`useComponentStore` 会自动降级到 `localStorage` 存储。开发者可直接在浏览器中操作所有功能，数据保存在浏览器本地。

### 7.2 postMessage 模拟

可在浏览器控制台中手动模拟主平台下发的数据：

```javascript

  type: 'platform-data',
  payload: {
    action: 'list-components',
    data: [
      {
        id: 'demo-1',
        name: 'Theme3LoanCalculator',
        description: '贷款额度计算器',
        version: '1.0.0',
        updatedAt: '2026-01-01T00:00:00Z',
        template: '<section class="...">...</section>',
        script: 'export default { ... }',
        style: '.theme3-calculator { ... }',
        i18nEntries: [],
        colorVars: [],
        bindings: [],
        eventBusName: '$EventBus'
      }
    ]
  }
}, '*')

// 模拟下发全局配置

    action: 'load-global-config',
```
