# 功能缺失分析与开发计划

> 基于项目完整代码审查（2026-06），列出三个核心模块的功能完成度与缺失项。

---

## 总体完成度

| 模块 | 完成度 | 核心功能 | 需要补齐的短板 |
|------|--------|----------|----------------|
| 搭建器 Builder | 45% | 拖拽放置 + 画布渲染 + 设备切换 | 布局无嵌套、无持久化、无导出页面配置 |
| 组件编辑器 ComponentEditor | 60% | 多语言 + 颜色 + 绑定 + 导出 + 沙箱 | 颜色未联动全局主题、AI 翻译未对接、版本缺失 |

| 跨模块协同 | 20% | postMessage + localStorage | 未实现完整生命周期、无错误处理 |

## 一、搭建器（Builder.vue）— 缺失清单

### 1.1 布局容器的嵌套功能（P0）
**现状**：布局容器（栅格/弹性/卡片）拖到画布后只显示图标占位，没有任何子元素承载能力。

**需要实现**：
- [ ] 栅格容器：拖入后可设置 `cols`（2/3/4 列），子模块自动分配到各列
- [ ] 弹性容器：支持 `direction`（row/column）、`wrap`、`justify`、`align`
- [ ] 卡片容器：带 header/body 区域，可拖入子模块
- [ ] 拖拽一个模块到布局容器上时 → 自动成为其子元素
- [ ] 嵌套层级指示（缩进线 / 面包屑导航）

**数据结构**：
```typescript
interface CanvasItem {
  instanceId: string
  componentId: string
  name: string
  type: 'layout' | 'module'
  layoutType?: 'grid' | 'flex' | 'card'
  children?: CanvasItem[] // 布局容器的子项
  props?: Record<string, any> // 布局参数
}
```

### 1.2 画布配置持久化（P0）
**现状**：`canvasItems` 仅在内存中，刷新丢失。'保存'按钮只发 postMessage 未落盘。

**需要实现**：
- [ ] 页面打开时通过 postMessage 向主平台请求上一次保存的配置
- [ ] 保存时序列化整个 `canvasItems` 树（含嵌套结构）
- [ ] 添加「另存为新页面」「加载页面」功能
- [ ] 页面配置数据结构：
```typescript
interface PageConfig {
  id: string
  name: string
  description?: string
  updatedAt: string
  device: string
  items: CanvasItem[]
}
```

### 1.3 拖拽体验增强（P1）
- [ ] 拖拽过程中显示插入位置指示器（水平线 + 箭头）
- [ ] 拖到布局容器上时高亮容器边缘
- [ ] 按住 `Ctrl` 拖拽 = 复制
- [ ] 右键菜单（编辑属性 / 复制 / 删除 / 移到容器）

### 1.4 撤销 / 重做（P1）
- [ ] 维护操作历史栈
- [ ] `Ctrl+Z` / `Ctrl+Shift+Z` 快捷键
- [ ] 工具栏撤销/重做按钮

### 1.5 导出页面为 JSON / HTML（P1）
- [ ] 「导出配置」按钮 → 生成完整的页面描述 JSON
- [ ] 「导出页面」按钮 → 生成一个独立 HTML 文件（内联所有组件）
- [ ] 可用于预览或部署

### 1.6 响应式配置（P2）
- [ ] 每个设备视图（mobile/tablet/desktop）可独立调整组件顺序
- [ ] 或指定某个组件在某个设备下隐藏

## 二、组件编辑器（ComponentEditor.vue）— 缺失清单

### 2.1 与全局配置的颜色联动（P0）
**现状**：颜色面板独立显示预设色，导出的 .vue 代码写死 `#2F6BFF`，未引用 GlobalConfig 的 7 色调色板。

**需要实现**：
- [ ] 组件编辑器的颜色面板读取 `GlobalConfig.themeColors` 的 7 个预设色
- [ ] 用户调整颜色后更新到组件模板中的 CSS 变量引用
- [ ] 导出的 .vue 改为 `var(--color-primary)` 等 CSS 变量引用
- [ ] 导出时附带 GlobalConfig 的 CSS 变量定义

### 2.2 沙箱预览响应语言切换（P0）
**现状**：沙箱内 Mock 的 `$t()` 只返回 key 本身，切换语言 Tab 不会更新预览内容。

**需要实现**：
- [ ] 沙箱 `$t()` 根据当前活跃语言返回对应的翻译值
- [ ] 切换语言 Tab → 触发 `$root.$emit('language-changed')` → 沙箱重新渲染

### 2.3 AI 翻译对接（P1）
- [ ] 替换占位按钮为实际 AI 翻译接口调用
- [ ] 支持「翻译当前条目」「翻译全部」两种模式
- [ ] 翻译后自动填入对应语言输入框

### 2.4 版本历史 / 草稿管理（P1）
- [ ] 每次「保存」生成一个新版本
- [ ] 文件列表中显示版本数，可展开查看历史
- [ ] 回滚到任意历史版本

### 2.5 导入 .vue 文件（P1）
- [ ] 「从文件导入」按钮 → 打开文件选择器
- [ ] 解析上传的 .vue 文件 → 自动提取 template/script/style
- [ ] 自动填充多语言条目、颜色、绑定信息

### 2.6 模板语法高亮（P2）
- [ ] 沙箱粘贴代码区域增加代码高亮
- [ ] 绑定显示时高亮对应元素

### 2.7 批量翻译引擎（P2）
- [ ] 一键翻译所有条目到所有语言
- [ ] 支持翻译记忆（已翻译的不重复翻译）

## 三、全局配置（GlobalConfig.vue）— 缺失清单

### 3.1 主题预设系统（P1）
**现状**：只有一套配色，无法切换不同主题。

**需要实现**：
- [ ] 内置 3~5 套预设主题（如「经典蓝」「翡翠绿」「暖阳橙」）
- [ ] 可保存当前配色为自定义主题
- [ ] 切换主题时联动其他模块

### 3.2 导出主题 CSS（P1）
- [ ] 「导出主题」按钮 → 生成完整的 CSS 变量文件
- [ ] 可供主平台直接引用

### 3.3 与组件编辑器的双向联动（P1）
- [ ] GlobalConfig 修改颜色 → 组件编辑器的颜色面板即时更新
- [ ] GlobalConfig 修改语言 → 组件编辑器的语言 Tab 即时增减

### 3.4 全局 i18n 字典管理（P2）
- [ ] 当前只有 key/value 管理，缺少分组/搜索/批量导入导出
- [ ] 支持导出为 JSON 字典文件
- [ ] 支持从 JSON 导入

## 四、跨模块与工程 — 缺失清单

### 4.1 postMessage 完整生命周期（P0）
**现状**：`usePostMessage.ts` 只有基础通讯方法，无完整的初始化/请求/响应协议。

**需要实现**（已在对接说明书中定义，需代码落地）：
- [ ] App.vue `onMounted` 时发送 `ready` 消息 → 主平台返回所有配置
- [ ] `save-page-config` / `save-component` / `list-components` / `delete-component` / `load-global-config` / `save-global-config` 6 条消息
- [ ] 统一错误响应格式：`{ type: 'error', message: string }`

### 4.2 统一加载状态与错误处理（P1）
- [ ] 所有异步操作（保存/加载/导入）增加 loading spinner
- [ ] 错误消息统一展示（toast 或 banner）
- [ ] 网络超时重试机制

### 4.3 弹窗/通知系统（P1）
- [ ] 当前使用 `alert()` / `confirm()` → 替换为自定义弹窗组件
- [ ] 成功/失败/信息三种通知类型

### 4.4 HMR 稳定性（P1）
- [ ] 解决 Vite 8 与沙箱环境的 HMR WebSocket 兼容性
- [ ] 自定义 HMR 路径需要和反向代理规则对齐

### 4.5 沙箱性能优化（P2）
- [ ] 当画布有 N 个模块组件时，同时运行 N 个 PreviewSandbox/ModulePreviewOnCanvas 实例
- [ ] 每个沙箱都创建一个独立的 Vue app (`createApp`)，内存开销大
- [ ] 优化：不可见的组件（在视口外）不做实时沙箱渲染，只显示静态截图
- [ ] 或：画布模块只渲染轻量预览，全功能预览统一在右侧面板

### 4.6 TypeScript 完善（P2）
- [ ] 当前多处使用 `as any` 绕过类型检查
- [ ] 沙箱 Mock 函数签名未完全对齐真实组件

### 4.7 单元测试（P3）
- [ ] 核心 composable 测试（usePostMessage, useComponentStore）
- [ ] 沙箱 parseSFC 函数测试

### 4.8 项目构建优化（P3）
- [ ] 生产构建 262KB（含全量 Vue 249KB），可考虑按需引入
